🚀 Panduan Setup Environment Variables

Supabase Bengkel System - Vercel Deployment

⚠️ PENTING: Environment variables harus di-set di Vercel Dashboard, bukan di kode!

📋 Langkah 1: Buka Vercel Dashboard

  1. Login ke Vercel Dashboard
  2. Pilih project Anda: supabase-aoo
  3. Klik tab Settings
  4. Klik menu Environment Variables
📸 Vercel Dashboard → Settings → Environment Variables

🔧 Langkah 2: Tambah Environment Variables

Tambahkan 2 variable berikut:

Variable 1:

Name: NEXT_PUBLIC_SUPABASE_URL
Value: https://efkkdeoheekcxwirungu.supabase.co
Environment: Production, Preview, Development

Variable 2:

Name: NEXT_PUBLIC_SUPABASE_ANON_KEY
Value: [Masukkan full anon key Anda di sini]
Environment: Production, Preview, Development
🔑 ANON KEY: Dapatkan full anon key dari Supabase Dashboard → Settings → API

💾 Langkah 3: Save dan Redeploy

  1. Klik Save setelah menambah setiap variable
  2. Setelah semua variable ditambah, klik Redeploy
  3. Tunggu proses redeploy selesai (2-3 menit)
  4. Refresh aplikasi Anda

✅ Langkah 4: Verifikasi Setup

Buka endpoint berikut untuk verifikasi:

Debug Environment:
https://your-app.vercel.app/api/env-check
Test Connection:
https://your-app.vercel.app/api/test-connection
✅ Jika berhasil, Anda akan melihat "success: true" dan data table

🔍 Troubleshooting

Jika masih error "Database tidak dikonfigurasi":

  • Pastikan environment variables tepat (case-sensitive)
  • Cek spelling: NEXT_PUBLIC_SUPABASE_URL (bukan SUPABASE_URL)
  • Pastikan anon key lengkap (dimulai dengan eyJhbGciOiJIUzI1NiIs...)
  • Redeploy lagi setelah menambah variables

Jika table tidak ditemukan:

  • Buat table di Supabase SQL Editor
  • Cek RLS policies di Supabase
  • Pastikan anon key memiliki akses read

🎉 Setup Selesai!

Setelah environment variables ter-set dengan benar, aplikasi akan menampilkan data dari Supabase database Anda.
